LLM 成熟框架讲解：中文讲义

核心主题：从大量 LLM 训练实践中归纳现代 Transformer 架构的共识、分歧、超参数默认值和训练稳定性技巧。

课程主线

这节课的标题是 Everything You Didn't Want To Know About LM Architecture And Training。它关注的不是 Transformer 的入门概念，而是训练现代 LLM 时真正会遇到的细节：架构变体、normalization、activation、position embedding、超参数、regularization、stability tricks，以及 attention 在推理时的系统成本。

讲者采用一种 data-driven / evolutionary analysis 的视角：过去几年已经有很多 LLM 被训练和发布，比较这些模型的共同点和差异，可以看出哪些设计是偶然选择，哪些已经成为现代 LLM 的稳定共识。

原始 Transformer 与现代共识版 Transformer

原始 Transformer 包含：

  • 底部的 position embeddings

  • multi-head attention

  • attention 和 MLP 后面的 LayerNorm

  • 向上贯穿的 residual stream

  • MLP / feed-forward network

  • 输出端 softmax

现代 LLM 的常见实现通常不是原始论文里的 vanilla Transformer，而更接近以下变体：

  • 把 normalization 放在 block 前面，即 pre-norm

  • 使用 rotary position embeddings / RoPE

  • feed-forward 层采用 SwiGLU 或其他 GLU variants

  • 大量 linear layers 去掉 bias terms

  • 越来越多模型加入额外稳定性设计，如 double norm, QK norm, z-loss

关键观点：现代 LLM 架构不是纯理论推导出来的，而是在大规模训练中通过经验逐渐收敛出来的。

Normalization：pre-norm、RMS Norm 与 bias-free

pre-norm vs post-norm

原始 Transformer 使用 post-norm：attention 或 MLP 计算后，先加回 residual stream，再做 LayerNorm。

现代 LLM 几乎都采用 pre-norm：先对输入做 LayerNorm 或 RMS Norm，再进入 attention/MLP，最后结果加回 residual stream。

主要事实与观点：

  • pre-norm 是现代 LLM 最强共识之一。

  • post-norm 训练更不稳定，常需要 careful learning rate warmup。

  • pre-norm 有利于梯度传播，避免 residual stream 中的 identity connection 被 normalization 破坏。

  • 讲者强调一个经验性“铁律”：normalization 最好放在 residual stream 外，而不是直接插在 residual stream 中。

  • 例外很少，讲者提到 OPT350M 是少数 post-norm 例子。

double norm / post-block norm

近期一些模型在 pre-norm 的基础上，又在 attention/MLP block 后加入 normalization。讲者暂称为 double norm。

事实：

  • Grok 和 Gemma 2 使用 block 前后都有 LayerNorm 的做法。

  • OLMo 2 采用只在 attention 和 feed-forward 后加 norm 的变体。

  • 这种设计被认为可以让大模型训练更稳定。

LayerNorm vs RMS Norm

LayerNorm 做两件事：减去均值、除以标准差，然后通过 learnable gamma 和 beta 缩放和平移。

RMS Norm 则去掉均值调整和 bias-like shift，只保留 RMS 尺度归一化。现代模型大量采用 RMS Norm。

关键 facts：

  • LLaMA family、PaLM、Chinchilla、T5 等都使用 RMS Norm。

  • RMS Norm 通常效果接近 LayerNorm，但更简单、更快。

  • 速度优势不是因为 FLOPs 大幅减少，而是因为减少 memory movement。

  • Ivanov et al. 2023 一类 profiling 结果显示：tensor contractions / matrix multiplies 占约 99.8% FLOPs，但 softmax、LayerNorm 等 normalization 操作虽然 FLOPs 很少，却可能占约 25% runtime，因为它们带来大量内存读写。

  • Narang et al. 2020 的实验中，vanilla transformer 约 3.5 steps/s，RMS Norm 约 3.68 steps/s，并且 final loss 更低。

关键词：

  • FLOPs

  • memory movement

  • tensor contraction

  • runtime bottleneck

  • normalization overhead

去掉 bias terms

现代 Transformer 常去掉 linear layers 中的 bias terms。

原因：

  • bias 对性能通常帮助不大。

  • 去掉 bias 可以减少参数和 memory movement。

  • 经验上，去掉 bias 有时能提升 optimization stability。

  • 现代大模型往往更接近“纯矩阵乘法”的结构。

Activation 与 gated MLP：SwiGLU 成为主流

早期模型使用 ReLU 或 GeLU：

  • 原始 Transformer 和一些旧模型使用 ReLU。

  • GPT-1、GPT-2、GPT-3、GPT-J 等使用 GeLU。

现代模型大量转向 gated linear units / GLU variants，尤其是 SwiGLU 和 GeGLU。

GLU 的核心思想

普通 MLP：

xxxxxxxxxx
MLP(x) = activation(xW1)W2

GLU-style MLP：

xxxxxxxxxx
MLP(x) = (activation(xW1) * xV)W2

其中 * 是逐元素乘法，xV 是 gate。直觉上，模型不仅把输入投影到 hidden space，还学习一个逐维 gate 来控制哪些 hidden features 通过。

SwiGLU 与 GeGLU

  • GeGLU：使用 GeLU 作为 activation，并带 gate。

  • SwiGLU：使用 Swish 作为 activation，并带 gate。

  • Swish 定义为 x * sigmoid(x)。

事实与观点：

  • PaLM、T5、Gemma 等 Google 系模型推动了 GLU variants 的普及。

  • 2023 年之后，几乎所有现代模型都使用 gated linear unit。

  • Noam Shazeer 的 GLU 论文和 Narang et al. 2020 的实验显示 GLU variants 往往有稳定收益。

  • GLU 不是训练好模型的必要条件。GPT-3 不用 GLU，Nemotron 340B 使用 squared ReLU，Falcon 211B 使用 ReLU，也能得到高性能模型。

  • 但经验上，SwiGLU / GeGLU 有持续收益，因此成为现代默认选择。

GLU 的 dff 缩放

GLU 额外引入 gate 参数 V，参数量会增加。为了让 GLU 模型和非 GLU MLP 参数量大致匹配，通常把 feed-forward hidden size 缩小到原来的 2/3。

常见规则：

  • 非 GLU MLP：dff = 4 * d_model

  • GLU / SwiGLU MLP：dff = 8/3 * d_model ≈ 2.67 * d_model

Serial layers vs parallel layers

标准 Transformer block 是 serial：先 attention，再 MLP，二者串行组合。

parallel layers 则让 attention 和 MLP 并行计算，然后一起加回 residual stream。

事实与观点：

  • GPT-J 开源复现工作较早使用 parallel layers。

  • PaLM 在大规模模型上使用了 parallel layers。

  • parallel layers 可带来系统效率：共享 LayerNorm、融合 matrix multiplications、提升 GPU utilization。

  • 但最近一年多数模型仍采用 serial layers。

  • 例外包括 Cohere Command A、Command R+、Falcon 2 11B。

  • 表达能力上，serial 可能更强，因为 attention 和 MLP 是组合关系；系统效率上，parallel 更有优势。

Position embeddings：RoPE 的胜出

早期位置编码有多种路线：

  • sinusoidal position embeddings：原始 Transformer 使用。

  • absolute learned position embeddings：GPT 系、OPT 等使用。

  • relative position embeddings：T5、Gopher 等使用。

  • ALiBi：曾经有一段时间被一些模型采用。

  • RoPE / rotary position embeddings：2023 年后成为事实标准。

RoPE 的核心目标

RoPE 强调 relative position。希望两个 token 的 attention 关系只依赖它们的相对距离 j - i，而不是绝对位置 i、j。

形式上，希望：

xxxxxxxxxx
<f(x, i), f(y, j)> = g(x, y, j - i)

即两个带位置的 embedding 内积，只依赖 token 内容和相对位置差。

RoPE 的直觉

RoPE 利用旋转的性质：向量同时旋转不会改变它们之间的相对角度和内积。把位置表示为旋转角度，token 在不同绝对位置会被旋转不同角度；两个 token 的内积自然编码相对位置差。

重要细节：

  • RoPE 不是在 embedding 底部加一个位置向量。

  • RoPE 作用在 attention 层中的 query 和 key 上。

  • RoPE 将高维向量切成多个 2D pair，每对维度按不同频率旋转。

  • 不同频率类似 sinusoidal embedding 的直觉：快频率捕捉局部/高频位置，慢频率捕捉远距离/低频位置。

  • RoPE 的角度 schedule 通常是固定的，不是训练出来的参数。

为什么 RoPE 成为主流：

  • 经验效果好。

  • 支持多种 context length extrapolation 技术。

  • 在小规模和短上下文上也有效。

  • 讲者调研的 19 个近期 dense model release 基本都使用 RoPE。

超参数选择：现代默认值

讲者强调：训练 LLM 时有很多 hyperparameters，但成功模型中真正变化的并不多，很多已经形成稳定经验默认值。

MLP hidden size / feed-forward dimension

核心参数：

  • d_model：residual stream / hidden state 的维度。

  • dff：MLP 内部 up-projection 后的 hidden dimension。

经验规则：

  • ReLU/GeLU-style MLP：dff = 4 * d_model

  • GLU/SwiGLU-style MLP：dff = 8/3 * d_model

例外：

  • T5 11B 使用非常夸张的比例：d_model = 1024，dff = 65000，约 64 倍。

  • Gemma 2 也使用较大比例，约 8 倍。

  • T5v1.1 后续回到更标准的 GeGLU 约 2.5 倍比例，并得到更好模型。

Kaplan scaling law paper 的相关实验显示，dff / d_model 在 1 到 10 之间有比较宽的低损失区间，4 是合理选择，但不是自然定律。

attention heads 与 head dimension

现代常见做法：保持总 hidden dimension d_model 不变，把它分给多个 heads。

即：

xxxxxxxxxx
d_model ≈ num_heads * head_dim

事实：

  • GPT-3、T5、LaMDA、PaLM、LLaMA2 基本都遵循这一比例约为 1 的规则。

  • T5 是少数例外，曾使用较大的比例。

  • 有研究担心 heads 太多、每个 head 维度太少会导致 low-rank bottleneck，但在实践中多数模型没有明显问题。

aspect ratio：深度与宽度

aspect ratio 这里指模型宽度和深度的关系，可以粗略理解为每层对应多少 hidden dimensions。

经验共识：

  • 很多 GPT-3 和 LLaMA-like 模型接近每层约 128 hidden dimensions。

  • Kaplan et al. 的实验显示，aspect ratio 约 100 附近在 50M、274M、1.5B 参数规模上都接近最优。

  • aspect ratio 的最优值似乎不会随模型规模大幅漂移，这是好消息，因为可以用固定比例扩展模型。

系统层面影响：

  • 更深模型适合 pipeline parallelism，把层切到不同设备。

  • 更宽模型适合 tensor parallelism，把矩阵切到不同 GPU。

  • tensor parallel 往往需要更高速网络；pipeline parallel 对网络延迟/带宽约束不同。

  • 因此模型宽深比不仅是表达能力问题，也是系统工程问题。

补充观点：

  • 上游 loss 上，参数量可能比深宽选择更重要。

  • downstream accuracy 可能不同，有研究认为同等 FLOPs 下更深模型可能有更好 downstream 表现。

vocabulary size

vocabulary size 呈上升趋势。

事实：

  • 早期模型，尤其偏单语模型，常在 30k 到 50k token vocab。

  • 现代 production / multilingual 系统常在 100k 到 250k token vocab。

  • GPT-4 tokenizer 及其被模仿者大约在 100k token 量级。

  • Cohere 等强调多语言的模型使用更大 vocab。

原因：

  • LLM 被部署到真实世界，需要支持多语言、emoji、特殊符号等。

  • 更大的 vocab 能让非英语、低资源语言被压缩成更少 token。

  • 对用户而言，少 token 意味着更低 inference cost 和更高吞吐。

  • 高资源语言如英语可以用较小 vocab 也工作良好；大 vocab 对低资源/非英语语言收益更明显。

dropout 与 weight decay

pre-training 通常只做一遍或不到一遍数据，理论上不太会过拟合，因此 dropout 逐渐不流行。

事实与观点：

  • 早期模型使用较多 dropout。

  • 现代 LLM pretraining 中 dropout 已经 out of fashion。

  • weight decay 仍然常见，但它的作用不主要是传统意义上的 regularization。

  • 实验显示，不同 weight decay 对 train-val loss gap 影响不大，因此不是在解决过拟合。

  • weight decay 似乎与 learning rate schedule 有复杂相互作用，尤其在 cosine learning rate decay 或训练末尾 learning rate 降到接近 0 时，能带来更好的 training loss。

  • 因此这里的 weight decay 更像是优化动态的一部分，而不是单纯防止 overfitting。

Stability tricks：现代模型的新重点

近一年核心架构变化不大，但很多新模型强调 training stability。随着模型更大、训练更久，loss spike、gradient norm explosion 等问题会更频繁。

稳定性问题集中出现在 softmax：

  • 输出端有 output softmax。

  • attention 内部有 attention softmax。

  • softmax 涉及 exponentials 和 normalization，数值上容易出问题。

z-loss

输出 softmax：

xxxxxxxxxx
p(x) = exp(logits) / z(x)

其中 z(x) 是 vocabulary 上的 normalizer。z-loss 的目标是让 z(x) 接近 1，即让 log z(x) 接近 0。

辅助 loss 形式：

xxxxxxxxxx
alpha * log^2 z(x_i)

事实：

  • Devlin 2014 的机器翻译工作中已有类似思想。

  • PaLM 在语言模型中使用 z-loss = 1e-4 * log^2 z，鼓励 softmax normalizer 更稳定。

  • Baichuan 2、DCLM、OLMo 2 等后续模型也采用 z-loss。

作用：

  • 稳定 output softmax。

  • 避免 softmax normalizer 过大或过小。

  • 让数值行为更接近稳定区域。

QK norm

QK norm 针对 attention softmax。做法是在 query 和 key 做 inner product 之前，先对 Q 和 K 进行 LayerNorm。

关键 facts：

  • QK norm 控制的是 softmax 输入的大小，而不是 normalizer z。

  • 起源于 vision / multimodal 模型训练稳定性实践。

  • Dehghani 2023 的大型 vision transformer 训练、Chameleon、IDEFICS 等使用过类似技巧。

  • 后续被 Gemma 2、DCLM、OLMo 2 等文本模型采用。

  • 推理时仍然保留 QK norm，因为 norm 有 learnable parameters，去掉会破坏模型行为。

  • NVIDIA 相关实验中，QK norm 可让模型使用更 aggressive learning rates，并改善 perplexity。

soft-capping logits

另一种方式是在 attention softmax 前对 logits 做 soft cap：

xxxxxxxxxx
soft_cap * tanh(logits / soft_cap)

作用：

  • 当 logits 远超 soft cap 时，tanh 会把值软截断。

  • 控制进入 softmax 的极端值。

事实：

  • Gemma 2、OLMo 2 采用过这种方法。

  • 它不如 QK norm 普遍。

  • 讲者提到 NVIDIA 实验中 soft-capping 反而让 perplexity 变差，而 QK norm 改善结果。

Attention heads 变体：MQA、GQA 与长上下文

训练和推理的 attention 成本差异

训练时，attention 使用大矩阵并行计算，GPU utilization 较好。推理时，自回归生成一次只能生成一个 token，需要依赖 KV cache。

KV cache：生成每个 token 时，把过去 token 的 keys 和 values 缓存起来。新 token 只需要计算当前 query 与过去 keys 的 attention。

核心问题：

  • 推理时 arithmetic operations 未必显著减少，但 memory access pattern 变差。

  • GPU 上 memory access 昂贵，compute 相对便宜。

  • arithmetic intensity 指每次内存访问对应多少计算，越高越好。

  • KV cache 推理中，K/V 的反复读取成为瓶颈，尤其长上下文时更明显。

MQA / Multi-Query Attention

MQA 的核心：保留多个 query heads，但所有 heads 共享一个 key head 和一个 value head。

优势：

  • 大幅减少 KV cache 需要存取的数据。

  • 提高推理时 arithmetic intensity。

  • 让更长 sequence length 更可行。

  • 对 inference cost 和 throughput 特别重要。

GQA / Grouped-Query Attention

GQA 是 MHA 和 MQA 之间的折中：

  • MHA：每个 query head 有自己的 K/V。

  • MQA：所有 query heads 共享一组 K/V。

  • GQA：多个 query heads 分成组，每组共享 K/V。

作用：

  • 在推理效率和模型表达能力之间折中。

  • 从 MHA 直接变到 MQA 可能太激进；GQA 通常更稳妥。

  • 一些工作显示 GQA 对模型质量影响较小，同时能带来推理收益。

长上下文 attention：sparse、sliding window 与混合模式

长上下文的核心矛盾：full attention 成本随 sequence length 二次增长。解决思路是改变 attention pattern。

历史思路：

  • OpenAI 2019 曾提出 sparse attention patterns。

  • GPT-3 使用过结构化/稀疏 attention 技巧来扩展 context window。

  • sliding window attention：每层只关注当前位置附近的局部窗口。

现代混合设计：

  • LLaMA 4、Gemma、Cohere Command A 等近期模型采用 block 级混合 attention。

  • 例子：每 4 个 Transformer blocks 中，底部 1 个使用 full self-attention 且无 position embedding；上面 3 个使用 sliding window attention with RoPE。

这个设计的好处：

  • 系统成本更低，因为 full attention 不在每层都做。

  • RoPE 只负责局部窗口，避免长距离 position extrapolation 困难。

  • 真正超长距离的信息通过无 position embedding 的 full attention 传播，理论上更容易 extrapolate 到极长上下文。

  • 讲者提到 LLaMA 4 宣称支持约 10 million tokens context，背后就涉及这类结构化 attention pattern。

总结：现代 LLM 架构的成熟共识

可以把现代 dense LLM 的默认架构概括为：

最重要的 takeaway：

  • pre-norm 几乎是铁律，核心原因是训练稳定性和梯度传播。

  • RMS Norm 因为效果接近、参数更少、memory movement 更少而成为主流。

  • bias-free linear layers 常见，既简化实现，也可能改善稳定性。

  • SwiGLU / GLU variants 提供稳定收益，是现代 MLP 默认选择。

  • RoPE 在 position embedding 竞争中胜出，尤其适合相对位置建模和 context extrapolation。

  • 超参数有强经验默认值：非 GLU dff=4*d_model，GLU dff≈8/3*d_model，d_model≈num_heads*head_dim，aspect ratio 约 100-128。

  • vocab size 从 30k-50k 增长到 100k-250k，主要服务生产、多语言和低资源语言。

  • dropout 在 pretraining 中不再流行，weight decay 仍常用，但更像优化动态技巧而非传统 regularization。

  • 近年的新重点是 stability tricks：z-loss、QK norm、soft-capping、double norm。

  • 推理效率推动了 KV cache、MQA、GQA、sliding window attention 等架构选择。
