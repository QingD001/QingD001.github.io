[ICPC 2001 Taejon R] 木棍加工

题目描述

一堆木头棍子共有 n 根，每根棍子的长度和宽度都是已知的。棍子可以被一台机器一个接一个地加工。机器处理一根棍子之前需要准备时间。准备时间是这样定义的：

  • 第一根棍子的准备时间为 1 分钟。

  • 如果刚处理完长度为 l，宽度为 w 的棍子，那么如果下一个棍子长度为 li，宽度为 wi，并且满足 l≥li,w≥wi，这个棍子就不需要准备时间，否则需要 1 分钟的准备时间。

计算处理完 n 根棍子所需要的最短准备时间。比如，你有 5 根棍子，长度和宽度分别为 (4,9),(5,2),(2,1),(3,5),(1,4)，最短准备时间为 2（按 (4,9),(3,5),(1,4),(5,2),(2,1) 的次序进行加工）。

思路分析

即对于 $ n $ 个元素的数对列，对 A(lA,wA),B(lB,wB) 定义偏序关系 ：

A⪯B⇔lA≤lB∧wA≤wB

求原数对列至少划分为多少个 ⪯ 意义上单调不增的子段？

根据 Dilworth 定理：有限偏序集中，最小链划分的大小等于其最大反链的大小。在这里，最大反链的大小等价于按其中一个坐标排序后，另一坐标的 LIS 大小。先按照如下比较函数排序：

bool cmp1(stick a,stick b){
    if(a.l!=b.l)
        return a.l>b.l;
    return a.w>b.w;
}

之后对维度 $ w $ 求 LIS 大小就行。一种做法是命dp[j]:以 j 结尾的 LIS 大小，时间复杂度 O(n2)；另一种是二分+贪心，维护一个 f[i]:长度为 i 的上升子序列的最小结尾元素 ，时间复杂度 O(nlog⁡n)。本题不需要输出具体序列，用第二种明显更优。

Code

xxxxxxxxxx
#include<bits/stdc++.h>
using namespace std;
const int N=5001;
int n,w[N];
struct stick{int l,w;}a[N];
bool cmp1(stick a,stick b){
    if(a.l!=b.l)
        return a.l>b.l;
    return a.w>b.w;
}
int main(){
    cin>>n;
    for(int i=1;i<=n;i++)
        scanf("%d %d",&a[i].l,&a[i].w);
    sort(a+1,a+n+1,cmp1);
    for(int i=1;i<=n;i++)
        w[i]=a[i].w;
    vector<int> f;
    for(int x:w){
        auto it=lower_bound(f.begin(),f.end(),x);
        if(it==f.end())
            f.push_back(x);
        else
            *it=x;
    }
    cout<<f.size()-1;
    return 0;
}

[CERC1998] 书的复制

题目描述

把 m 本有顺序的书分给 k 个人复制（抄写），每一个人的抄写速度都一样，一本书不允许给两个（或以上）的人抄写，分给每一个人的书，必须是连续的，比如不能把第一、第三、第四本书给同一个人抄写。 现在请你设计一种方案，使得复制时间最短。复制时间为抄写页数最多的人用去的时间。

思路分析

把 a1~am 划分成 $ k $ 个连续子段，使每段和的最大值最小

本题的初始状态明确，考虑顺推。对当前环境的最小充分描述可用当前划分的段数 i 和处理到的书编号 j 充分描述，边界条件为 f1,j=sj 其中 sj=∑i=1jai 。

计算 fi,j 时，我们枚举上一段的末尾书本编号 $ l $，显然有状态转移方程：

fi,j=minl=1,2,...,j−1{max(fi−1,l,sj−sl)}

输出时要求后面的人尽量多抄，自然地想到递归。

Code

xxxxxxxxxx
#include<bits/stdc++.h>
using namespace std;
const int N=501;
int m,k,a[N],s[N],f[N][N];
void print(int i,int j){ //前 i 本书给 j 个人抄的划分方案
    if(!j)return ;
    if(j==1){
        printf("1 %d\n",i);
        return ;
    }
    int p=i,q=a[i];
    while(q+a[p-1]<=f[k][m]){
        q+=a[p-1];
        --p;
    }
    print(p-1,j-1);
    printf("%d %d\n",p,i);
}
int main(){
    cin>>m>>k;
    for(int i=1;i<=m;i++)
        cin>>a[i],s[i]=s[i-1]+a[i];
    memset(f,0x3f,sizeof(f));
    for(int i=1;i<=k;++i){
        for(int j=1;j<=m;++j){
            f[1][j]=s[j];
            for(int l=1;l<j;++l)
                f[i][j]=min(f[i][j],max(f[i-1][l],s[j]-s[l]));
        }
    }
    print(m,k);
    return 0;
}

[ICPC 1993 WF] 加油问题

题目描述

一个美国旅行代理商经常被要求去估计开车从一个城市旅行至另一个城市的最小费用。他有一个在通常路线上的大多数加油站的列表。列表包括了所有加油站的位置及当前每加仑汽油的价格。

为了简化估计费用的过程，代理商使用了以下的简化汽车驾驶员行为的规则：

  • 除非汽车无法用油箱里的汽油达到下一个加油站（如果有的话）或目的地，在油箱里还有不少于最大容量一半的汽油时，驾驶员从不在加油站停下来。

  • 在每一个停下的加油站，驾驶员总是将油加满。

  • 在一个加油站停下之后，驾驶员将为旅程在快餐和糖果上花去 2.00 元。

  • 在驶向加油站或目的地时，驾驶员不需要超过必须量的汽油。不需要“安全余量”。

  • 驾驶员开始旅行时油箱总是满的

  • 每个加油站付款时四舍五入到分（ 1 元等于 100 分）。

你必须写一个程序以估计驾驶员在旅程上至少要为汽油和食品付多少钱。

思路分析

初始状态明确，考虑顺推。对当前环境的充分描述可通过 (pos,oil) 实现，分别代表当前所处的加油站编号和剩余油量；然而注意到规则 2，油必然会被加满，所以这一维度可以被压缩掉；而对下一个加油站是否需要停留/如果停留该怎么更新完全可以通过计算得知。这题相比于前两题，可以说很简单了。

Code

xxxxxxxxxx
#include<bits/stdc++.h>
using namespace std;
const int N=52;
const double INF=1e18;
double a[N],b[N],f[N];
int n;
int main(){
    double s,g,x,c;
    cin>>s>>g>>x>>c>>n; //总距离、邮箱容量、每升能行驶的距离、初始加油的钱
    for(int i=1;i<=n;++i)
        cin>>a[i]>>b[i];
    a[n+1]=s,a[0]=0; //0为起点,n+1为终点,1~n 为加油站
    double lims=g*x;
    for(int i=0;i<=n+1;i++)
        f[i]=INF;
    f[0]=c;
    for(int i=0;i<=n+1;i++){
        if(f[i]==INF)
            continue;
        for(int j=i+1;j<=n+1;++j){
            double dist=a[j]-a[i];
            if(dist>lims)break;
            double tmp=g-dist/x;
            if(j==n+1)
                f[j]=min(f[j],f[i]);
            else
                if(tmp<g/2)
                    f[j]=min(f[j],f[i]+2.00+(dist/x)*b[j]/100.0);
        }
    }
    printf("%.2lf",f[n+1]);
    return 0;
}

[USTCPC2026] 下午茶时光

题目描述

在某个悠闲的下午，Furai 面前摆着 $ n $ 块精致的小点心，排成一列。第 i 块点心的“甜蜜值”为 ai。Furai 可以任选一些点心来组合（也可以一个都不选）。

把选中的点心下标按从小到大排序后，显然会形成若干段连续的区间组合。例如：选中点心的下标 2,3,4,7,8，会形成两段区间组合：[2,4] 和 [7,8]；选中点心的下标 5,6,7，就是一段区间组合 [5,7]；如果一个都不选，则没有区间组合。 对于每一个区间组合 [L,R] 中，Furai 会重新按顺序给每个点心一个编号，也就是 1,2,…,R−L+1，她只打算认真品尝编号为奇数的点心（获得对应的“甜蜜值”），而编号为偶数的点心只是为了让组合看起来更丰富，实际上她并不吃它们（因此不计“甜蜜值”）。

每次开始品尝一个新的组合，Furai 都需要花费 C 的“甜蜜值”来调整心情。 这次下午茶时光的总“甜蜜值”是所有区间组合的“甜蜜值”之和。 虽然 Furai 肯定吃不完这么多点心，但是她还是希望知道能得到的最大“甜蜜值”。

思路分析

第一个洞察是：按照顺序选和随便选是等价的，因为区间必须是连续的才可能对结果产生贡献。进一步分析，由于存在贡献的区间必须连续，我们完全没必要关心具体区间是怎么切分的，只要按顺序处理每个甜点 i ，看它是没被选上/选上了但是奇数/选上了但是偶数。

具体而言，定义 fi,0/1/2 分别对应上述三种情况。显然 fi,0=maxj=02{fi−1,j} ；而对于 fi,1，说明 i 应该被选了且为奇数编号，也就是说 i−1 的时候有两种可能：原先段的偶数/原来没被选，新开一段。对于 fi,2 ，它只能继承 fi−1,1 的值，此外任何情况都不能对它产生贡献。

最终结果对 fn,0/1/2 中取 max 产生。

Code

xxxxxxxxxx
#include<bits/stdc++.h>
using namespace std;
const int N=3e5+10;
typedef long long LL;
const LL INF=1e18;
int n;
LL c,a[N],f[N][3];
int main(){
    cin>>n>>c;  
    for(int i=1;i<=n;i++)
        cin>>a[i];
    f[0][1]=f[0][2]=-INF;
    for(int i=1;i<=n;i++){
        f[i][0]=max(max(f[i-1][0],f[i-1][1]),f[i-1][2]);
        f[i][1]=max(f[i-1][0]+a[i]-c,f[i-1][2]+a[i]);
        f[i][2]=f[i-1][1];
    }
    LL ans=max(max(f[n][0],f[n][1]),f[n][2]);
    cout<<ans;
    return 0;
}

[ICPC 2017 WF] Posterize

题目描述

数字图像中的像素可以用三个范围在 0 到 255 之间的整数表示，分别表示红、绿、蓝三种颜色的强度。为了压缩图像或创造艺术效果，许多照片编辑工具包括一个 posterize 操作，其工作原理如下。每个颜色通道单独检查；这个问题只关注红色通道。对于红色通道，posterized 图像最多允许 k 个整数，而不是允许从 0 到 255 的所有整数。每个像素的原始红色强度被替换为允许整数中最接近的一个。照片编辑工具选择一组 k 个整数，以最小化原始图像中所有像素引入的平方误差之和。如果有 n 个像素的原始红色值为 r1,…,rn，并且有 k 个允许的整数 v1,…,vk，则平方误差之和定义为

∑i=1nmin1≤j≤k(ri−vj)2

你的任务是计算在给定参数 k 和图像像素红色强度描述的情况下，可以实现的最小平方误差之和。

思路分析

给定整数集合 R={r1,r2,…,rd}⊆[0,255]∩N，每个 ri 对应权重 pi∈\N 与整数 k。要求选择一个集合 S⊆[0,255]∩N ，满足 |S|=k ，定义 S 的代价函数：

c(S)=∑i=1dpi⋅minv∈S(ri−v)2

要计算 minc(S)

集合往往是难以处理的，我们把它转化成连续的区间来看。等价于将 (r1,p1),(r2,p2)…(rd,pd) 划分到 k 个连续区间，每个区间 [L,R] 的代价为：

CL,R=minv∈Z∑i=LRpi⋅(ri−v)2

其中 v 限制在 min{rL,…,rR} 到 max{rL,…,rR} 之间。 总代价即为各区间代价之和。这样，我们其实只要关心这 k 个区间到底怎么切分即可。定义 dp[i][j] 表示前 i 个 $ r $ 划分到 j 个区间的最小总代价，显然有：

dpi,j=min0≤t<i{dpt,j−1+Ct+1,i}

其中 Cl,r 可以通过预处理区间加权和与加权平方和快速求解。

Code

xxxxxxxxxx
#include<bits/stdc++.h>
using namespace std;
const int N=267;
typedef long long ll;
const ll INF=1e18;
int main(){
    int n,k;
    cin>>n>>k;
    vector<pair<int,int>> a(n);
    for(int i=0;i<n;++i)
        cin>>a[i].first>>a[i].second;
    sort(a.begin(),a.end());
    vector<int> pos(n+1),cnt(n+1);
    for(int i=1;i<=n;i++)
        pos[i]=a[i-1].first,cnt[i]=a[i-1].second;
    vector<ll> sum1(n+1,0),sum2(n+1,0),sum3(n+1,0);
    for(int i=1;i<=n;i++)
        sum1[i]=sum1[i-1]+cnt[i],sum2[i]=sum2[i-1]+1ll*pos[i]*cnt[i],sum3[i]=sum3[i-1]+1ll*pos[i]*pos[i]*cnt[i];
    vector<vector<ll>> cost(n+1,vector<ll>(n+1,INF));
    //cost[l][r] 代表 [l,r] 内所有人去同一个代表点的最小代价
    for(int l=1;l<=n;++l){
        cost[l][l]=0;
        for(int r=l+1;r<=n;++r){
            ll s=sum1[r]-sum1[l-1],sr1=sum2[r]-sum2[l-1],sr2=sum3[r]-sum3[l-1];
            ll res=INF;
            for(int x=0;x<=255;++x){
                ll cur=sr2-2ll*sr1*x+1ll*x*x*s;
                res=min(res,cur);
            }
            cost[l][r]=res;
        }
    }
    vector<vector<ll>> dp(n+1,vector<ll>(k+1,INF)); //dp[i][j] 前 i 个点分为 j 段的最小代价
    dp[0][0]=0;
    for(int i=1;i<=n;++i){
        for(int j=1;j<=min(i,k);++j){
            for(int t=0;t<i;++t)
                if(dp[t][j-1]<INF)
                    dp[i][j]=min(dp[i][j],dp[t][j-1]+cost[t+1][i]);
        }
    }
    ll ans=INF;
    for(int i=1;i<=k;++i)
        ans=min(ans,dp[n][i]);
    cout<<ans;
    return 0;
}
