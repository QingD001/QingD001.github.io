Introduction to Byte Latent Transformer

Local Encoder

考虑将输入的长度为 N 的自然语言按照 UTF-8 编码转化成原始字节 (Bytes) 序列：

X={x1,x2,...,xN},xi∈{0,1,...,255}

这种设计的优势在于：

  • UTF-8 编码下，相似的文字往往具有相似字节前缀

  • 模型直接读取字符，不再需要学习词整体，通过权重转移自然学习生成概率

  • 理论上可以表示任何二进制数据

转化成字节后，序列长度扩大 4 倍，注意力计算成本扩大 16倍，这显然不可接受

为了阐述的简便，本文按照以固定长度 K （BLT论文的原版先进策略是按照信息熵动态切分）将字节序列切分成 M=⌈N/K⌉ 个不相交 Patch ，最后一行空余部分由占位符 [pad] 填充的形式讲解：

Pj=[x(j−1)K+1,...,xjK]∈ZK,j=1,2,...,M

整个输入数据 X 的形态从 (N)→(M,K)，即：

[P1P2...PM]=[ x1 , x2 , x3 , ... , xK xK+1,xK+2,xK+3, ... ,x2K... x...,xN,[pad], ... ,[pad]]

行代表一个 Patch ，列代表 Patch 内相对位置。

实际处理时还有一维度 B 代表 batch size ，为书写简便此处不表。

符号声明：

需要知道的是，每个 Patch 在计算时并行发生、互不干扰。

字节嵌入层

每个 Pt(t∈1,2,...,M) 还是一串数字序列，这些编码本身不具有语义。我们期望它具有语义的特征，故需要嵌入层。

因此考虑训练一个小的“词典/表”（线性变换）W∈R256×d，其初始是随机的。

我们可以通过查词典获得Pt 每个元素 λ 对应的向量 vλ，具体而言：

PAt=[85,32,97,72]→查表变换PAtA′=[vA85,vA32,vA97,vA72]AT

（查表操作本质上是将每个位置的元素拆成 one-hot 向量后作乘法）

经过本层后，数据形态呈现为 (M,K,d) 的张量，这为后续的卷积和注意力计算定下数学维度基础。

词典将在反向传播中产生“意义”，让语义相似的形态也相近，提高嵌入质量。

局部位置编码层

由于接下来的卷积核是共享参数的，同一个卷积核 J 在处理第 i 个字节时用的权重和处理第 j (j≠i) 个字节时用的权重并无不同。

如果不添加位置编码：

  • 模型读取 [v1,v2] 和 [v2,v1] 得到的特征分布可能会非常相近，而它们对应的自然语义可能天差地别。

  • Patch 内部结构固定，无法确定哪个向量该送去池化

因此，可以给每个位置 i 分配一个独有的向量 posi，直接加到 Embedding 上。这样它们在嵌入空间中被强行拉开距离，且带有独特标记。

LayerNorm

考虑到尺度漂移的问题，经过多层乘法/加法的复合变换，数值分布差异巨大。

由于激活函数的设计，数值极大或极小时梯度接近于0，模型可能忽略部分有效信息；优化器不知道该大步还是小步，训练震荡剧烈。

本层使得每一层的输出向量都拉回到标准分布，对于：

μ=1d∑i=1dxi   σ2=1d∑i=1d(xi−μ)2

使得：

x^i=xi−μσ2+ϵ

其中 ϵ 是一个极小值如 10−5，防止除以零的 Runtime Error。

然而这样做的后果就是数据被锁死为正态分布，模型的表达能力受限，因此考虑引入两个可学习的参数：

  • γ ：控制缩放

  • β ：控制平移

这两个参数都是向量，长度同 d。我们令：

yi=γi⋅x^i+βi

这不仅维持了数值稳定，还保留了模型自定义分布的能力。

局部特征提取层

由于前述中 Patch 的计算并行发生且互不干扰，我们只考虑 (K,d) 的部分

这一阶段期望：让时间序列上每个位置的向量，都“看见”它先前的部分信息并融合起来。

由于预测任务的时间序列特性，“未来”（即该行后面的部分）的数据是不能看的；因此我们需要对卷积核进行改造，让它具有“因果”特性：

  • Padding (补零)：在矩阵顶部补 0

  • Masking (掩码)：确保卷积核只对过去和现在加权

具体实现来说，可以分为如下几个Step：

Step 1 : 权重初始化

准备形如 (din,dout,k)的卷积核 J

考虑到输入接口对齐与残差链接，应该有：

din=dout=d

当前行 t 的输出是由时刻 t−(k−1), ... ,t−1,t 的信息共同计算出的

  • J[: , : , 0]：负责当前行 t 的信息

  • J[: , : , 1]：负责前 t-1 时刻的信息

  • ...

  • J[: , : , k-1]：负责前 t-(k-1) 时刻的信息

（如果在 Pytorch 框架下实现代码，这个对应顺序应该反过来）

Step 2 : Padding

为使第 0→(k−2) 行参与卷积运算，要在输入矩阵顶端补上 k−1 个零行向量。

补完后的输入矩阵记为 Xpad。

Step 3 : 滑动窗口计算

大小为 k 的卷积核，在 Xpad 上从上往下滑动，滑一步算一次：

Yt=∑i=0k−1(Xt−i⋅J[:,:,i])+b

其中 b 为偏置量。

完整滑动一次后，我们得到了序列 Y0,Y1,...,YK−1 。

Step 4 : 激活函数

为了让模型能拟合更复杂的场景，需要对每一行的每个元素过一遍激活函数：

Yfinal[t]=GELU(Yt)

这样引入了非线性，提高模型性能。（其它激活函数也可）

期望让 Patch 内最后一个向量“集成”整个 Patch 的信息，本部分总共需要 p 层。

根据感受野计算公式，有：

1+p(k−1)≥K

池化层

这是 Local Encoder 部分的最后一阶段，意在通过前面的操作将一个 Patch 压缩成一个向量。

由于 Patch 内最后一个向量几乎集成了整个 Patch 的信息，故考虑 Last-token Pooling ，这种池化方式是检索任务中高性能的做法。

Mean-Pooling 会平均化 Patch 内所有位置的向量。由于因果卷积的特性，Patch 内前 K−1 个位置的感受野尚未覆盖完整 Patch。将这些信息不全的中间状态与信息完整的最终状态作平均，会引入噪声并稀释关键特征。所以一般不用。

投影层

如果隐藏层维度和输出维度不一致，可以添加本投影层。

上述即为 Local-Encoder 的架构概览。

Global Transformer

Global Transformer 由多个相同层构成，我们以第 l 层（约定 l∈{1,2,, ... ,L}为例，仍然以数据流转的视角分析层内结构。在本部分开篇，首先明确一下符号规范：

本章节具体计算中，仍先忽略 B 这一维度。

预归一化

H=LayerNorm(Hl−1)

Linear Projection & Multi-Head Split

首先将 Latent Embeddings 映射到注意力的 Q,K,V 空间。记 WQ,WK,WV∈Rd×d 为可学习权重矩阵，

Q=H(l−1)WQK=H(l−1)WKV=H(l−1)WV

此时 Q,K,V∈RM×d 已对内容信息进行初步建模。然而如果只有一组WQ,WK,WV，模型在同一个空间对语义、语法、指代进行建模，容易混淆；因此采用 Multi-Head 的形式，将大空间拆分成多个子空间让模型进行不同角度的理解。

具体而言，将最后一个维度 d 拆分成 (h,dk)，并交换维度以便并行计算：

Qsplit,Ksplit,Vsplit∈Rh×M×dk

接下来操作，都是针对每一个头而言的。

旋转位置编码RoPE

在 BLT 的设计中，位置编码必须反映原始字节序列级别的距离信息。固定长度切分的情形下，第 m 个 Patch 对应的字节位置索引为（这里为 0-based 索引）：

idxm=(m−1)×K

定义旋转算子 R(idx):Rd→Rd，它将向量 v 在 d/2 个独立的二维平面上进行旋转。

具体而言，对于向量 qm 的第 i 对维度 (2i,2i+1)，旋转操作定义为：

[q~m,2iq~m,2i+1]=[cos⁡((m−1)K⋅θi)−sin⁡((m−1)K⋅θi)sin⁡((m−1)K⋅θi)cos⁡((m−1)K⋅θi)][qm,2iqm,2i+1]

对于 km 同样的操作：

[k~m,2ik~m,2i+1]=[cos⁡((m−1)K⋅θi)−sin⁡((m−1)K⋅θi)sin⁡((m−1)K⋅θi)cos⁡((m−1)K⋅θi)][km,2ikm,2i+1]

其中频率 θi=10000−2i/d，如果先前未了解过RoPE位置编码，读者可以先行思考为何此处仅对 q,k 进行旋转操作。而 Q~,K~ 为旋转后向量聚合的矩阵。

注意力计算

根据注意力公式：

Scores=Q~K~Td

由于 BLT 的自回归预训练架构，还需要因果掩码Mask∈RM×M：

Maski,j=if (i≥j) ? 0:−∞

最终的注意力分数：

A=softmax(Mask+Scores)

最终加权求和，用注意力权重取出 Value 内容：

O=A⋅V

所有头分别并行计算完成后，所有 O 就会聚合回 (B,M,d) 的形状。

残差链接

为何防止深度网络梯度消失，模型应该保留原始信息，数学上来看即：

Z1=O+H(l−1)

进行一次归一化以稳定数值分布：

Z2=LayerNorm(Z1)

前馈神经网络

将归一化后的 Z2 送入 MLP（多层感知机），通常包含两个线性变换和一个非线性激活函数得到：

Z3=FFN(Z2)

其中：

FFN(x)=(xW1+b1)⋅Activation⋅W2+b2

将结果 Z3 再次加回状态 Z1，并再次归一化，得到 H(l)：

H(l)=LayerNorm(Z3+Z1)

经过上述操作，训练时模型收敛速度、计算速度、信息保留度、数值分布稳定程度都得以在一定程度上被考量。

Prediction Heads

经过 Global Transformer 部分后，模型形成了 H(L) 的张量，它是一个压缩储存完整上下文本的容器。本部分将改张量重新转回字节级输出。

State Classifier

模型首先要判定当前的生成状态

  • 输入：当前 Patch 对应的状态 ht∈H(L)

  • 功能：输出一个二分类概率，决定下一步的操作模式：

    • Continue：当前 Patch 还没结束，直接输出 Patch 内下一字节

    • Predict：当前 Patch 已结束，需要借助语义信息预测新 Patch 第一个字节

数学形式：

P(mode)=σ(ht⋅Wgate+bgate)

其中 Wgate 和 bgate 为可训练的权重矩阵与偏置向量。

Content Generation Head

当 State Classifier 决定需要预测新字节时，本部分才启动工作：

  • 输入：和 State Classifier 同样的 ht

  • 功能：预测下一个字节的具体值（0-255）

数学形式：

Logitsnew=ht⋅Wgen+bgen

其中 Wgen∈Rd×256 和 bgen 为可训练的权重矩阵与偏置向量。

生成概率：

P(xnext|mode=Predict)=Softmax(Logitsnew)

Length Prediction Head（可选）

在某些 BLT 变体（譬如 BLT-entropy）中，模型还需要预测下一个 Patch 应该多长。它的结构和前面二者相似，这里略去不表。
